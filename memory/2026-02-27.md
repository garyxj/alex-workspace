## 核心观点

**这不是安装指南**，而是如何管理和使用OpenClaw的指南。很多人安装后很快放弃，因为不知道如何管理它。

## 安装后的四项协议（Post-Install Protocols）

### A. "数字永生"协议（GitHub集成）
**问题：** 默认情况下，重装服务器会丢失所有记忆

**解决方案：** 连接到专用GitHub仓库，作为代理的长期记忆
- 提交每日日志
- 从中拉取"标准操作程序"（SOPs）
- **结果：** 即使服务器崩溃，"大脑"已备份到云端

### B. 每日"站会"（经验总结）
在每个操作周期结束时，让OpenClaw总结"经验教训"并写入记忆：
- 今天哪些网站阻止了我？
- 哪些提示语法产生了最好的代码？

随着时间的推移，这个文件成为复利资产。代理变聪明不是因为AI模型升级，而是因为**经验**增长。

### C. 心跳检查（60分钟检查）
自主代理的最大风险之一是"漂移"。**解决方案：**
- 严格的心跳规则
- 通过Telegram/Slack/Discord每30-60分钟报告状态
- 强制代理"反思"自己的进度

如果意识到在相同任务上卡了两个周期，它可以自我纠正。

### D. "临时身份"策略
**绝不要给OpenClaw你的个人凭证！**
- 不要登录个人Gmail
- 不要给主LinkedIn账户
- 把OpenClaw当作数字世界中的独立实体

**做法：**
- **命名它：** 给它一个独特的身份
- **给它凭证：** 创建专用的Gmail（如 `agent.nomos.01@gmail.com`）和burner GitHub账户

这不仅是安全问题，也是**上下文分离**。

## "不要在家尝试"警告

**不要在家庭电脑上运行OpenClaw！**

当你给自主代理访问互联网和执行终端命令的能力时，你实际上把房子的钥匙交给了陌生人。

**风险：**
- OpenClaw下载恶意包或与受损网站交互
- 它可以访问你的本地文件系统
- 可能意外删除家庭照片或暴露SSH密钥

**解决方案：** 使用云
- 花5美元/月
- 在DigitalOcean、AWS或Hetzner上启动虚拟专用服务器（VPS）
- 在云端Docker容器中运行OpenClaw

如果它失控？**删除服务器。** 如果它被感染？**删除服务器。** 你的个人笔记本保持干净。

## 安全政策实施

### 问题
敏感信息（API 密钥、密码、账户凭证）被提交到 GitHub，存在安全风险。

### 解决方案
创建 .gitignore 文件，保护敏感信息：
- **忽略的文件类型：**
  - API Keys: api-keys.md, *_api_key*, *_apikey*, *_token*
  - 密码和凭证: passwords.md, accounts.md, secrets.md, *_credentials.md
  - 环境配置: .env, .env.local, .envrc, .env.production, .env.staging

### 执行步骤
1. 创建 `.gitignore` 文件
2. 从 git 历史中移除 `memory/agent-credentials.md`（使用 `git rm --cached`）
