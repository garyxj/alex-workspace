## 用例一：自然语言 CRM - 0分钟从零到可用

**核心能力：**
- 371 个联系人，全部可以用自然语言查询
- 关系健康评分，自动标记长期没联系的人
- 重复联系人检测和合并建议
- 向量嵌入搜索，支持语义级别的模糊匹配

**工作流程：**
1. **数据摄取**
   - 系统每 30 分钟扫描邮件
   - 每 5 分钟在工作时段检查 Fathom（AI 会议记录工具）
   - 所有数据存入前先经过一个 LLM 判断：这封邮件值不值得存？这个联系人重不重要？

2. **关键特性：**
   - **自我学习过滤**：如果 Berman 拒绝了某条行动项（「这不算我的任务」），系统会学习原因并更新提取规则
   - **每天自动检查 3 次完成情况**：比如 Berman 在会上说「我今天把邮件发过去」，系统会检查他是否真的发了，发了就自动打勾

3. **跨模块联动：**
   - 最惊艳的细节：当 Berman 在其他场景（比如想视频选题）时，CRM 会主动插嘴：
   「你之前和某个赞助商聊过类似话题，也许他们愿意赞助这期。」
   - 系统跨模块联动，不只是被动存数据，而是主动创造连接

**价值：**
- 30 分钟搭建完全定制的 CRM
- 如果 30 分钟再花两小时迭代优化，为什么要付费给 CRM 公司？

## 用例二：会议行动项自动追踪

**工作流：**
1. **会议结束 → 自动提取行动项**
   - 会议记录全文 → OpenClaw 匹配 CRM 系统
   - 提取行动项 → 发送到 Telegram → Berman 审批
   - 审批通过的自动进 Todoist

2. **关键设计：**
   - **区分「我的」和「对方的」行动项**
     - 对方承诺要给你的东西，系统标记为「waiting on」，自动追踪对方是否兑现
   - **自我学习过滤**
     - 如果 Berman 拒绝了某条（「这不算我的任务」），系统会学习原因并更新提取规则
   - **每天自动检查 3 次完成情况**
   - **14 天自动归档**：过期未完成的自动清理，保持清单干净

**价值：**
- 把「会议后续」这个最容易烂尾的环节完全自动化

## 用例三：个人知识库，丢个链接进去就行

**数据类型和处理：**
- **文章**：直接抓取全文，对付费墙网站用浏览器自动化登录后提取
- **YouTube 视频**：抓取字幕转文本
- **X 帖子**：不只抓单条，会自动追踪整个帖子串，外链文章一并摄入
- **PDF**：直接解析文本

**所有内容做向量化嵌入，存入本地 SQLite**

**搜索能力：**
- 可以用自然语言搜索：「给我看所有关于 OpenAI 的文章」
- 一键检索

**团队协作：**
- 每条入库内容自动以「Matt 想让你们看看这个」的形式同步到 Slack
- 团队知道这是老板亲自读过的，不是 AI 随便推的

**使用门槛：** 极低
- 不需要打标签、不需要分类、不需要整理
- 丢进去就行，AI 帮你做剩下的

**价值：**
- 从「灵感捕捉」到「可执行方案」的距离压缩到了接近零

## 用例四：商业顾问委员会，8个专家每晚帮你开

**数据输入：** 4 个业务数据源
- YouTube 分析
- Instagram 每帖互动
- X 分析
- TikTok 数据
- 邮件活动
- 会议记录
- Cron 任务健康状况
- Slack 消息

**分析流程：**
1. 多个 AI 专家角色（财务、营销、增长、运营等）各自独立分析全部数据
2. 分析完成后它们互相讨论发现，综合分歧
3. 合并成一份按优先级排序的建议清单
4. 每天凌晨 3:30 自动运行

**交付方式：**
- 结果以编号摘要发到 Telegram
- Berman 起床后扫一眼，对任何一条都可以追问：「展开说说第 3 条」

**启发性：AI Agent 协作模式**
- 不是一个 AI 给你建议
- 而是一群 AI 互相辩论后给你建议
- 就像真正的董事会，财务说要省钱，营销说要花钱，最后折中出一个务实方案

## 用例五：安全委员会，AI 每晚审查 AI

**架构设计：**
- **运行时间：** 每晚凌晨 3:30（错开其他任务的时段，避免 Anthropic API 配额冲突）
- **审查团队：** 4 个维度的安全专家
  - 进攻视角
  - 防御视角
  - 数据隐私视角
  - 操作真实性视角
- **审查范围：** 整个代码库、Git 提交历史、运行日志、错误日志、存储数据
- **审查方式：** 不是静态规则扫描，是让 AI 真正读代码并理解逻辑

**输出：** Opus 4.6 综合所有发现，编号后发到 Telegram

**自我进化机制：**
- 每次修复后的经验会被记住
- 审查规则持续迭代
- 有些夜晚没有新建议，因为系统确认当前状态是安全的

**最妙的设计：**
- 让 AI 来审 AI 自身
- 提示注入的防护永远不可能完美
- 但与其假装风险不存在，不如让系统每天体检一次

## 用例七：视频选题流水线，从一句话到 Asana 卡片

**触发方式：**
- 在 Slack 讨论中，任何人在帖子下面回复「[@Claude](https://x.com/@Claude)，这是一个视频创意」

**自动化流程：**
1. 读取 Slack 讨论串的完整上下文
2. 全网搜索 + X 趋势调研
3. 查询知识库，看是否有相关已存素材
4. 查重，检查是否和已有选题重复
5. 生成完整视频大纲：
   - 标题建议
   - 缩略图建议
   - 开场 hook
   - 视频流程框
6. 做一个「这个选题值不值得做」的评估
7. 在 Asana 创建项目卡片，附带所有研究资料和链接

**演示案例：**
- Quen 3.5 发布的消息被分享到 Slack
- 有人标记为视频创意，系统自动生成了完整的选题包
  - 包括 Twitter 上不在 KOL 的讨论
  - 开源社区的反应
  - 以及建议的视频角度

**价值：**
- 把「灵感捕捉」到「可执行方案」之间的距离压缩到了接近零

## 用例八：记忆系统，让 AI 越用越懂你

**记忆层级：**

1. **对话记忆**
   - 每日对话自动保存为 markdown 文件

2. **偏好提炼**
   - 从对话中提取：
     - 写作偏好
     - 语气风格
     - 关注兴趣
     - 股票追踪
     - 邮件分类规则
   - 存入 `memory.md`

3. **身份更新**
   - 每次新对话开始时，系统读取记忆文件
   - 更新 `identity.md` 和 `soul.md`

4. **向量化检索**
   - 所有记忆文件向量化，支持 RAG 搜索

**场景化人格切换：**
- Berman 给 OpenClaw 配了两套性格
  - **Telegram 私聊：** 像朋友，幽默随意
  - **Slack 团队频道：** 自动变成专业同事风格

**关键转变：**
- 这个用例让 AI 从「工具」变成了「伙伴」
- 它不只是执行指令，而是真正理解你是谁、你要什么

**价值：**
- 用越久越懂你
- 不同场景自动切换人格，无缝衔接

## 用例九：食物日记，AI 帮你发现过敏

**使用方式：**
- 拍食物照片发给 OpenClaw
- 它自动识别并记录
- 每天收 3 次提醒，汇报胃部感受
- 所有数据存入食物日志

**分析能力：**
- 每周触发一次分析
- 交叉比对食物记录和症状报告
- 识别模式

**真实成果：**
- 系统通过分析照片中的食物成分和 Berman 的症状反馈
- 发现他的胃对洋葱敏感
- 这完全是他自己完全不知道的

**价值：**
- 一个聊天机器人，帮人做了食物过敏原排查
- 在以前需要去医院做专项检测

## 用例十：定时任务 + 自动备份 + 自动更新

**Cron 任务清单：**

| 频率 | 任务 |

|------|------|

| 每 5 分钟 | 检查 Fathom 会议记录 |

| 每 30 分钟 | 扫描邮件 |

| 每天 3 次 | 行动项完成检查 |

| 每晚 | 文档同步、CRM 扫描、安全审查、日志摄取、视频数据刷新、晨间简报生成 |

| 每周 | 记忆合成、收益预测 |

| 每小时 | Git 提交 + 数据库备份 |

**备份策略：**
- 所有的 SQLite 数据库自动发现、加密、打包上传到 Google Drive，保留最近 7 天
- 代码每小时 Git 推送到 GitHub
- 任何备份失败，Telegram 立即告警

**自动更新：**
- 每天 9 点检查 OpenClaw 新版本
- 展示 changelog
- 一句「update」就自动升级重启

**API 追踪：**
- 记录每次 LLM 调用了哪个模型、消耗多少 token
- 甚至下载了各模型官方的 prompting 指南
- 让系统根据实际使用的模型优化 prompt 写法

**设计理念：**
> 「你睡觉的时候，系统在干活；系统出问题的时候，你第一时间知道。」

**这套基础设施的设计理念就一条：你睡觉的时候，系统在干活；系统出问题的时候，你第一时间知道。」

## 回到全局：这些用例之间的关系才是重点

### 数据飞轮效应

如果你只看单个用例，会觉得「挺酷，但好像也没那么特别」。

ChatGPT 也能帮你查联系人，Notion AI 也能帮你整理知识库。

**Berman 系统的真正威力在于用例之间的数据流：**
- CRM 数据 → 喂给商业顾问委员会
- 知识库内容 → 喂给视频选题流水线
- 社交媒体数据 → 喂给每日简报 + 顾问委员会
- 顾问委员会 → 会议记录 → 喂给 CRM + 行动项系统
- 所有模块的运行日志 → 喂给安全委员会 + 系统更新

**每个模块都不是孤岛。它们构成了一个互相强化的数据飞轮。**

> 「你会开始看到我搭建的所有不同部分是如何相互作用、让彼此变得更强大的。」

这才是为什么一个人 + 一台 MacBook 能产出一个小公司中台团队的效果。

## 安全提醒：跑之前先系好安全带

### Matthew Berman 在安全上下的功夫

**提示注入防御：**
- 所有外部内容视为潜在恶意，入库前确定性代码预扫描
- 输出控制：总结不逐字复述，自动过滤密钥和令牌
- 发布审批：发邮件、发推文前必须人工确认
- 加密备份：双重密码保护，.env 文件永不入库

### 他自己说得很清楚：

> 「没有完美的安全方案。大语言模型是非确定性系统，完全防住提示注入是不可能的。但这不意味着你什么都不做。」
