### 问题

- 不原生支持 Tavily API

### 解决方案
- 创建了自定义 Tavily 搜索技能：`C:\Users\jia.xu\.openclaw\skills\tavily-search\`
- 技能包含 `tavily_search` 工具，支持所有 Tavily API 参数

### 配置
- Tavily API key: `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`
- 存储位置:
  - `memory/agent-credentials.md` (备份)
  - `C:\Users\jia.xu\.openclaw\.env` (环境变量)
- 技能已添加到 `openclaw.json` 的 `skills.allowBundled` 列表

### 技能文件
- `SKILL.md` - 技能文档
- `index.ts` - 技能实现 (TypeScript)
- `package.json` - 包信息
- `README.md` - 使用说明

### 工具参数
- `query` (required): 搜索查询
- `search_depth`: "basic" 或 "advanced"
- `max_results`: 1-100 (默认 10)
- `include_domains`: 限制域名
- `exclude_domains`: 排除域名
- `days`: 搜索过去 N 天
- `topic`: "general" 或 "news"

### 经验教训
- OpenClaw 的 web_search 配置只支持 `brave` 和 `perplexity` provider
- 要使用其他搜索服务，必须创建自定义技能
- 技能通过 `process.env` 读取环境变量是推荐方式
- 技能需要在 `openclaw.json` 的 `skills.allowBundled` 中启用

### 用户偏好
- 优先使用 Tavily Skill (tavily_search) 作为搜索工具
- 仅在 Tavily 不可用时使用 web_search
