## C. 心跳检查（漂移监控）

### 配置文件
- `heartbeat-monitor.md` - 心跳检查配置文档

### Cron Job
- **名称**: Heartbeat Check
- **ID**: 4b65e24a-6e88-4858-8fbf-3e7fd26e5b16
- **时间**: 每30分钟
- **状态**: 已禁用（测试后启用）

### 功能
- 防止代理漂移
- 定期状态检查和自我反思
- 自动纠正机制

### 状态报告格式
```
🐾 爪爪心跳报告

当前任务：[任务名称]
进度：[完成度 %]
时间：[已用时间 / 预计时间]
阻碍：[如果有]

反思：
✅ 在正确轨道上 / ❌ 需要调整方向

下一步：[下一个行动]
```

### 安静时间规则
- 23:00 - 08:00：不发送心跳（除非紧急）
- 周末：减少频率为每小时一次
- 只在以下情况报告：
  - 工作时间（09:00-23:00）
  - 发现紧急问题
  - 任务完成

## 故障排查

### GitHub备份失败
- 检查远程仓库访问权限
- 确认网络连接
- 查看git错误日志

### 每日站会未生成
- 检查memory目录权限
- 确认cron job是否运行
- 查看agent日志

### 心跳检查过于频繁
- 调整cron表达式
- 修改安静时间规则
- 减少报告频率
