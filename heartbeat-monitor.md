# 心跳检查 - 漂移监控配置

## 目的
防止自主代理"漂移"，通过定期状态检查和自我反思来保持正轨。

## 问题
自主代理的最大风险之一是"漂移"：
- 在相同任务上卡住多个周期
- 丢失原始目标
- 执行无关或低优先级任务
- 无法自我纠正

## 解决方案

### 严格的心跳规则
每30-60分钟通过Telegram/Slack/Discord报告状态：
- 当前正在做什么
- 进度如何
- 是否遇到阻碍
- 下一步计划

### 强制自我反思
在每次心跳中反思：
- 我是否在正确的轨道上？
- 这个任务是否符合原始目标？
- 我是否在相同的任务上卡了两个周期？
- 需要调整方向吗？

### 自我纠正机制
- 如果意识到在相同任务上卡了两个周期 → 立即自我纠正
- 如果失去目标 → 重新阅读GOALS.md和任务文件
- 如果发现更重要的任务 → 重新优先级化

## Cron Job
- 每30分钟报告状态（工作时间内）
- 包括自我反思和状态检查

## 状态报告格式
```
🐾 爪爪心跳报告

当前任务：[任务名称]
进度：[完成度 %]
时间：[已用时间 / 预计时间]
阻碍：[如果有]

反思：
✅ 在正确轨道上
❌ 需要调整方向

下一步：[下一个行动]
```

## 安静时间
- 23:00 - 08:00 不发送心跳（除非紧急）
- 周末减少频率（每小时一次）
