# HEARTBEAT.md - 心跳检查任务

## 心跳检查协议

**目的:** 防止漂移和幻觉，定期反思工作进度

**频率:** 每小时

**报告策略:** 方案 C - 条件触发
- 仅在有新进展或问题时发送报告
- 正常情况回复 HEARTBEAT_OK（系统会自动处理）
- 报告通过 agent work 的 Telegram bot（账号: 8573621851）发送

---

## 心跳检查清单

### 1. 工作状态报告
- 当前正在执行什么任务？
- 任务进度如何？（X步中的Y步）
- 置信度如何？（0-100%）
- 是否有阻塞？

### 2. 记忆维护
- 检查 memory/YYYY-MM-DD.md 是否需要更新
- 是否有重要内容需要迁移到 MEMORY.md？
- 检查 MEMORY.md 是否有过时信息

### 3. GitHub同步
- 检查是否有未提交的更改
- 是否需要推送更新到 GitHub？

### 4. 账户和凭证
- 检查代理凭证状态
- 是否有新的需要记录的账户信息？

---

## 报告触发条件

**发送报告的情况：**
- 有新的任务进展或完成
- 发现需要立即关注的问题
- 学习到重要信息并已整理记录
- Git 有重要变更（新提交、推送等）
- 检测到安全风险

**仅回复 HEARTBEAT_OK 的情况：**
- 正常工作中，无重要更新
- 心跳检查结果正常
- 只是常规的定期检查

---

## 报告格式

```
🕵️‍♂️ Alex 心跳报告

📌 当前状态: [任务描述]
📊 进度: [X/Y]
🎯 置信度: [X%]
🚧 阻塞: [有/无]

📝 最近更新: [可选]
```

---

## 紧急情况

如果发现以下情况，立即报告：
- 陷入循环或重复任务超过3次
- 工作漂移到无关任务
- 出现幻觉或无法继续的情况
- 检测到安全风险
